# Use Case: Публикация товара

## Актор
Продавец маркетплейса

## Цель
Разместить товар на витрине маркетплейса для дальнейшей продажи.

## Предусловия
- Пользователь авторизован
- Пользователь зарегистрирован как продавец
- Пользователь имеет доступ к личному кабинету продавца

## Основной сценарий

1. Продавец открывает раздел "Мои товары"

2. Нажимает кнопку "Добавить товар"

3. Система отображает форму создания товара
    - Обязательные поля помечаются специальным индикатором

4. Продавец заполняет обязательные поля:
   - Название товара
   - Описание
   - Категория
   - Цена
   - Остаток товара

5. Продавец загружает изображения товара

6. Продавец нажимает кнопку "Опубликовать"

7. Система выполняет валидацию данных

8. Система сохраняет товар

9. В зависимости от правил модерации маркетплейса, система присваивает товару статус:
   - "На модерации"
   или
   - "Опубликован"

10. Система отображает сообщение об успешном создании товара


## Альтернативные сценарии

### 1. Сохранение черновика

1. Продавец заполняет часть данных
2. Нажимает кнопку "Сохранить как черновик"
3. Система сохраняет товар со статусом "Черновик"

### 2. Редактирование товара

1. Продавец открывает существующий товар
2. Изменяет информацию
3. Нажимает кнопку "Сохранить"
4. Система обновляет данные товара


## Ошибки и исключения

### 1. Не заполнены обязательные поля

- Система отображает сообщение об ошибке рядом с незаполненными полями
- Обязательные поля выделяются визуально
- Публикация товара невозможна
- Пользователю предлагается "дозаполнить обязательные поля", отмеченные ярким цветом для быстрого нахождения
- Проверка заполнения обязательных полей может выполняться при вводе данных или при попытке публикации товара

### 2. Загружен неподдерживаемый формат изображения

- Система отображает ошибку загрузки файла
- Пользователю предлагается загрузить другое изображение

### 3. Слишком большой размер изображения

- Система отображает сообщение о превышении допустимого размера файла
- Пользователю предлагается загрузить изображение меньшего размера


### 4. Ошибка сохранения товара

- Система отображает сообщение:
  "Не удалось сохранить товар. Попробуйте позже."
- Введенные пользователем данные сохраняются в форме
- Пользователю предлагается повторить попытку публикации

## Постусловия

- Товар сохранен в системе
- Товар доступен продавцу в списке товаров
- Товар может отображаться покупателям в каталоге после публикации

# Use Case: Изменение статуса товара

## Актор
Продавец маркетплейса

## Цель
Изменить статус товара для управления его отображением на витрине маркетплейса.

## Предусловия
- Пользователь авторизован
- Пользователь имеет доступ к личному кабинету продавца
- Товар уже создан

## Основной сценарий

1. Продавец открывает список своих товаров

2. Выбирает нужный товар

3. Изменяет статус товара:
   - "Активен"
   - "Скрыт"
   - "Отсутствует в наличии"

4. Система сохраняет новый статус товара

5. Система обновляет отображение товара на витрине маркетплейса

6. Система отображает сообщение об успешном изменении статуса

## Ошибки и исключения

### 1. Ошибка сохранения статуса

- Система отображает сообщение:
  "Не удалось изменить статус товара. Попробуйте позже."

- Пользователю предлагается повторить попытку

## Постусловия

- Статус товара обновлен
- Отображение товара обновлено в каталоге маркетплейса

# Use Case: Удаление товара

## Актор
Продавец маркетплейса

## Цель
Удалить товар из каталога маркетплейса.

## Предусловия
- Пользователь авторизован
- Пользователь имеет доступ к личному кабинету продавца
- Товар существует в системе

## Основной сценарий

1. Продавец открывает список товаров

2. Выбирает нужный товар

3. Нажимает кнопку "Удалить"

4. Система отображает подтверждение удаления

5. Продавец подтверждает удаление товара

6. Система удаляет товар из каталога

7. Система отображает сообщение:
   "Товар успешно удален"

## Альтернативные сценарии

### 1. Отмена удаления

1. Продавец нажимает кнопку "Отмена"
2. Система закрывает окно подтверждения
3. Товар остается без изменений


## Ошибки и исключения

### 1. Ошибка удаления товара

- Система отображает сообщение:
  "Не удалось удалить товар. Попробуйте позже."

- Пользователю предлагается повторить попытку


## Постусловия

- Товар удален из каталога маркетплейса
- Товар больше не отображается покупателям

# Нефункциональные требования

1. Личный кабинет продавца должен быть доступен в поддерживаемых регионах работы маркетплейса.

2. Система должна обеспечивать защиту от несанкционированного доступа.

3. Система должна поддерживать загрузку изображений популярных форматов:
   - JPG
   - PNG
   - WEBP

4. Система должна поддерживать загрузку изображений максимального размера: 10 МБ.

5. Система должна поддерживать максимальное количество изображений в карточке: до 30 штук. 

6. Время публикации товара, а также время сохранения любых изменений в карточке товара не должно превышать 5 секунд при нормальной нагрузке.

7. Система должна сохранять введенные данные при возникновении ошибок публикации товара.

8. Интерфейс должен корректно отображаться
в актуальных версиях современных браузеров:
- Google Chrome
- Mozilla Firefox
- Safari
- Microsoft Edge
- Яндекс.Браузер
